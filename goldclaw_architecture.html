🦁 GoldClaw V3.2 Architecture Overview

A cybernetic control loop connecting edge devices (IronClaw agents) with a central orchestrator (Mothership) via the AMCP protocol over CloudEvents. The system is designed for interoperability, idempotency, and resilience using Goldrush pre-filtering.

Data Flow

Protocol: CloudEvents 1.0 over HTTP
Normalization: Edge (1.0) ↔ Cloud (1.0.2)
Filtering: CPU > 90% OR Status ≠ "online"
Idempotency: Instruction leasing with 300s TTL
📡

Edge Agent (IronClaw)

Rust/WASM runtime executing tasks in a secure sandbox.

Heartbeat → Sends CPU/memory/status every minute (with jitter)

← Instruction Poll

Result → Returns execution output

🔍

Pre-Filter (Goldrush)

Intelligent event filtering to prevent noise.

Significant Event → Dispatch to Fleet Dispatcher

Normal Heartbeat → Drop silently

🚀

Fleet Dispatcher

Routes signals to appropriate Shadow agents.

Agent ID Extraction → Parse "urn:ironclaw:agent:UUID"

Get or Start Agent → Dynamic lookup

Forward Signal → Async GenServer.cast

📦

Instruction Queue

Manages commands to edge agents.

Lease → Atomic assignment (300s TTL)

Reaper → Expired lease cleanup (60s)

Idempotency → Duplicate result acceptance

Technology Stack

Mothership: Elixir/OTP 28
Framework: Jido 2.0
Event Stream: Goldrush (develop-elixir branch)
Database: PostgreSQL (Ecto)
Web: Plug/Cowboy
Edge: Rust/WASM
Protocol: CloudEvents 1.0
Security: HMAC-SHA256
Deployment: Fly.io

📝 Examples Folder

Located at /Users/speed/.openclaw/workspace/edge-config/, this folder contains:

  • jido_uplink.json — Heartbeat routine configuration
  • instruction_poller.json — Instruction polling and execution loop
  • .env.example — Environment variable template

These JSON files demonstrate the exact configuration format that the IronClaw edge agent would use to communicate with the GoldClaw Mothership. They include:

  • Cron triggers with jitter_sec to prevent thundering herds
  • HMAC signature headers (X-Cybernetic-Timestamp, X-Cybernetic-Nonce, X-Cybernetic-Signature)
  • CloudEvents 1.0 compliant JSON payloads
  • HTTP POST/GET actions for heartbeats, instruction fetching, and result submission
